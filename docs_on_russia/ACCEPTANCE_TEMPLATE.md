

## 3. Smoke-тесты

Проверяйте только то, что относится к текущей задаче.

- Backend импортируется успешно.
- Health endpoint возвращает OK.
- Docker Compose config валиден, если Docker существует.
- Миграция БД выполняется, если есть Alembic.
- Upload endpoint работает только после задачи загрузки.
- Реальный внешний LLM-вызов не требуется, если задача явно этого не запрашивает.

---

## 4. Ручная проверка

Убедитесь:

- нет изменений в `docs/SDD.md`, кроме первоначальной копии из источника;
- нет секретов в diff;
- нет масштабных переписываний вне рамок задачи;
- routes остаются тонкими;
- доступ к хранилищам осуществляется через клиенты/репозитории;
- LLM-вызовы, когда они появляются, идут только через `LLMGateway`;
- все модели документов/чанков/фактов включают или планируют `access_level`;
- заглушки/моки задокументированы.

---

## 5. Ожидаемый git diff

diff должен содержать:

- файлы, перечисленные в задаче;
- минимальные вспомогательные файлы;
- тесты или smoke-проверки, где уместно;
- обновлённый `docs/HANDOFF.md`.

diff не должен содержать:

- `.env`;
- реальные учётные данные;
- не связанные переписывания frontend/backend;
- реализации будущих задач;
- сгенерированные кэш-файлы;
- локальные файлы IDE.

---

## 6. Решение о коммите

Коммитить только если:

- Definition of Done выполнен;
- проверки проходят или неблокирующие проблемы задокументированы;
- `docs/HANDOFF.md` обновлён;
- `git diff` ограничен по объёму;
- секреты отсутствуют.

git add <файлы>
