# TASK_007_text_parsers

## Цель

Реализовать MVP извлечение/парсинг текста для TXT, Markdown, PDF, DOCX, CSV и XLSX.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-006
- SDD требования к загрузке документов и парсингу

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/services/parsing/__init__.py`
- `backend/app/services/parsing/base.py`
- `backend/app/services/parsing/text_parser.py`
- `backend/app/services/parsing/markdown_parser.py`
- `backend/app/services/parsing/pdf_parser.py`
- `backend/app/services/parsing/docx_parser.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Предоставить интерфейс парсера, возвращающий извлечённый текст плюс базовые метаданные.
- Поддерживать `.txt`, `.md`, `.pdf`, `.docx`, `.csv`, `.xlsx` на уровне MVP.
- Сохранять подсказки страницы/секции/листа, где практично.
- Обрабатывать ошибки парсинга явно и безопасно.
- Использовать небольшие тестовые фикстуры или сгенерированные временные файлы.

---

## Явно не делать

Не делать:

- разбивать текст на чанки
- сохранять чанки
- извлекать сущности/факты
- индексировать поиск
- вызывать LLM
- реализовывать OCR
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
