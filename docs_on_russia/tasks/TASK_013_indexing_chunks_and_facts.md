# TASK_013_indexing_chunks_and_facts

## Цель

Индексировать документы, чанки, сущности и факты в Elasticsearch после приёма.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-012
- SDD требования гибридного поиска и трассируемости

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/search/indexing_service.py`
- `backend/app/services/ingestion/indexing_step.py`
- `backend/tests/unit/test_indexing_service.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Строить Elasticsearch документы из PostgreSQL записей.
- Индексировать метаданные документа, чанки, сущности, факты, числовые условия.
- Включать ID источников и `access_level` в каждый индексируемый элемент.
- Использовать bulk индексацию, где практично.
- Сделать индексацию устойчивой к повторным попыткам/идемпотентной на уровне MVP.

- реализовывать API запросов

- вызывать LLM
- генерировать синтез ответов

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
