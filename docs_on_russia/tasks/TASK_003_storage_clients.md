# TASK_003_storage_clients

## Цель

Добавить базовые модули backend клиентов/фабрик сессий для внешних систем хранения:

- PostgreSQL
- Redis
- Elasticsearch
- Neo4j
- MinIO

Цель — только конфигурация и безопасное создание. Не реализовывать доменное сохранение, индексацию, запись графа или workflow загрузки.

- Neo4j — слой графовых обходов.

- Redis — поддержка очередей/кэша/статусов задач.

## Требования

Реализовать минимальные вспомогательные конструкции:

- Фабрика SQLAlchemy engine/session для PostgreSQL или async эквивалент, основанный на текущем стиле проекта.
- Фабрика клиента Redis.
- Фабрика клиента Elasticsearch.
- Фабрика драйвера Neo4j.
- Фабрика клиента MinIO.

Health route может показывать здоровье приложения и опционально статус конфигурации зависимостей, но не должен падать только потому, что локальные внешние сервисы не работают, если это не явно спроектировано как deep health.

Модули клиентов должны:

- читать из централизованных настроек;
- избегать логирования секретов;
- не подключаться с eager-подключением, которое ломает импорт приложения;
- быть тестируемыми с моками или тестами только на конструирование.

---

## Явно не делать

Не делать:

- создавать SQLAlchemy модели;
- создавать Alembic миграции;
- реализовывать репозитории документов;
- реализовывать upload endpoint;
- реализовывать создание bucket MinIO, если это не безвредный helper;
- создавать маппинги Elasticsearch;
- писать ограничения Neo4j;
- добавлять Celery задачи;
- добавлять LLMGateway;
- добавлять frontend;
- изменять `docs/SDD.md`.

## Команды валидации

Из корня репозитория:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если Docker сервисы доступны:

```bash
docker compose up -d postgres redis minio elasticsearch neo4j
```

Но тесты не должны требовать живых сервисов, если не помечены как интеграционные.

## Definition of Done

- Существуют модули клиентов хранилищ.
- Приложение импортируется без живых внешних сервисов.
- Фабрики клиентов используют настройки.
- Unit-тесты валидируют поведение конструирования/конфигурации без реальных секретов.
- Health endpoint остаётся доступным.
- Нет преждевременно реализованной логики доменного сохранения/индексации/графа/загрузки.
- `docs/HANDOFF.md` обновлён Gemma.
