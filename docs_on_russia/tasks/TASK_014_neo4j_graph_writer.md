# TASK_014_neo4j_graph_writer

## Цель

Записывать MVP узлы и отношения графа знаний в Neo4j из извлечённых сущностей/фактов.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-013
- SDD требования к графовой навигации и отношениям

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/graph/__init__.py`
- `backend/app/graph/schema.py`
- `backend/app/graph/graph_writer.py`
- `backend/app/services/ingestion/graph_step.py`
- `backend/tests/unit/test_graph_writer.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Создать ограничения/индексы для MVP меток узлов.
- Записывать базовые узлы: Document, Chunk, Material, Process, Equipment, Property, Organization, Location.
- Записывать отношения с метаданными исходного факта/чанка/документа.
- Включить `access_level` или метаданные доступа, где требуется для фильтрации обхода.
- Сделать записи идемпотентными с использованием стабильных ключей.

---

## Явно не делать

Не делать:

- реализовывать UI графа
- реализовывать API запросов графа сверх helper-тестов
- вызывать LLM
- выводить отношения без доказательств
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
