# TASK_010_numeric_extractor_and_units

## Цель

Извлекать числовые условия и нормализовать единицы измерения детерминированно.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-009
- SDD числовые условия и принцип детерминированного извлечения

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/services/nlp/numeric_extractor.py`
- `backend/app/services/nlp/unit_normalizer.py`
- `backend/app/repositories/facts.py`
- `backend/app/models/fact.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Извлекать температуру, pH, давление, концентрацию, скорость потока, производительность, CAPEX/OPEX, процент извлечения, год.
- Сохранять сырое значение/единицу/текст и нормализованное значение/единицу.
- Прикреплять метаданные исходного документа/чанка/диапазона.
- Создавать кандидатные факты с метаданными детерминированного метода извлечения.
- Покрывать русскую и английскую нотацию, где практично для MVP.

---

## Явно не делать

Не делать:

- вызывать LLM
- выводить пропущенные значения
- писать финальные проверенные факты без статуса кандидата/проверки
- реализовывать frontend UI проверки
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
