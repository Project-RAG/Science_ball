# TASK_009_dictionary_entity_extraction

## Цель

Извлекать MVP доменные сущности с использованием детерминированных словарей и алиасов.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-008
- SDD типы сущностей и принцип "LLM не является источником истины"

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/services/nlp/__init__.py`
- `backend/app/services/nlp/dictionaries.py`
- `backend/app/services/nlp/entity_extractor.py`
- `backend/app/repositories/entities.py`
- `backend/app/models/entity.py`
- `backend/tests/unit/test_entity_extractor.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Поддерживать MVP типы сущностей: Material, Process, Equipment, Property, Organization, Location.
- Использовать словари/алиасы и детерминированное сопоставление.
- Возвращать spans, нормализованное имя, тип сущности, ID исходного чанка.
- Сохранять извлечённые сущности через репозиторий.
- Держать данные словарей небольшими и редактируемыми.

---

## Явно не делать

Не делать:

- вызывать LLM
- обучать модели
- реализовывать полный UI онтологии
- писать граф Neo4j
- индексировать Elasticsearch, если не требуется существующими абстракциями
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
