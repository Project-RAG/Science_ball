# TASK_011_confidence_service

## Цель

Реализовать MVP оценку доверия (confidence scoring) для извлечённых фактов и сущностей.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-010
- SDD confidence, evidence, принцип ограничения LLM

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/services/nlp/confidence_service.py`
- `backend/app/schemas/confidence.py`
- `backend/tests/unit/test_confidence_service.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Определить детерминированную формулу confidence на основе метода извлечения, качества доказательств, типа источника, успешности нормализации.
- Представить правило, что извлечение только через LLM не может превышать настроенный лимит confidence.
- Хранить входные данные confidence или объяснение, где практично.
- Держать оценку настраиваемой и тестируемой.

---

## Явно не делать

Не делать:

- вызывать LLM
- реализовывать workflow проверки
- широко изменять существующие извлечённые факты без тестов
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
