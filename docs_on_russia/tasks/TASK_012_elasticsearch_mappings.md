# TASK_012_elasticsearch_mappings

## Цель

Создать управление индексами Elasticsearch и маппинги для документов, чанков, сущностей, фактов и векторов.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-011
- SDD требования Elasticsearch к поиску/векторам/числам

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/search/__init__.py`
- `backend/app/search/index_names.py`
- `backend/app/search/mappings.py`
- `backend/app/search/index_manager.py`
- `backend/tests/unit/test_elasticsearch_mappings.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Определить маппинги для документов, чанков, сущностей, фактов.
- Включить `access_level` в каждый поисковый документ.
- Представить числовые поля/диапазоны и вложенные числовые условия.
- Сделать размерность вектора конфигурируемой.
- Предоставить идемпотентный helper создания/обновления индекса.

---

## Явно не делать

Не делать:

- индексировать реальные данные пока
- генерировать эмбеддинги
- реализовывать поисковый API
- вызывать LLM
- изменять `docs/SDD.md`
- коммитить реальные секреты или приватные данные

---

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```

---

## Definition of Done

- Маппинги валидируются как plain dicts в тестах.
- Index manager может быть протестирован с мокированным Elasticsearch клиентом.
- `access_level` существует во всех маппингах.
- `docs/HANDOFF.md` обновлён Claude Code с Gemma-4-31B.
