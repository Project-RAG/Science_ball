# TASK_008_chunking_service

## Цель

Реализовать детерминированное разбиение текста на чанки и сохранить чанки с трассируемостью документа.

---

## Входной контекст

Прочитать перед кодированием:

- `docs/SDD.md`
- `docs/AI_RULES.md`
- `docs/HANDOFF.md`
- файлы предыдущих задач по применимости
- этот файл задачи

Соответствующий контекст:

- файлы предыдущих задач 001-007
- SDD требования к трассируемости и чанкам

---

## Файлы для создания или изменения

Ожидаемые файлы:

- `backend/app/services/ingestion/chunking_service.py`
- `backend/app/repositories/chunks.py`
- `backend/app/schemas/chunks.py`
- `backend/app/models/chunk.py`
- `backend/tests/unit/test_chunking_service.py`

Добавлять минимальные вспомогательные файлы только когда требуется импортами/тестами.

---

## Требования

- Разделять распарсенный текст детерминированно по настраиваемому размеру/перекрытию.
- Сохранять чанки с `document_id`, порядком/индексом, текстом, исходной страницей/секцией, если доступны.
- Хранить `access_level` на чанках.
- Сохранять сырой текст доказательств трассируемым к исходному документу.
- Сделать разбиение идемпотентным для того же документа/задачи, где практично.

## Команды валидации

Из корня репозитория, адаптировать под текущий этап:

```bash
cd backend
python -m pytest
python -m compileall app
```

Если изменены frontend файлы:

```bash
cd frontend
npm install
npm run build
```

Если требуются Docker сервисы и они доступны:

```bash
docker compose up -d
```
